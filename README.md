# HPC 集群监控面板

 Slurm 算力集群的网页监控工具。

## 准备工作：安装 Node.js

本工具依赖 Node.js 运行环境。如果电脑尚未安装，请按以下步骤操作：

### 下载安装包

1. 打开浏览器访问 **https://nodejs.org**
2. 点击绿色的 **LTS**（长期支持版）按钮下载 Windows 安装包（`.msi` 文件）

### 安装步骤

1. 双击下载的 `node-vxx.x.x-x64.msi` 安装包
2. 一路点 **Next**（下一步），**不需要修改任何选项**
3. 安装程序会自动勾选 **「Add to PATH」**（添加到环境变量），保持默认即可
4. 点击 **Finish** 完成安装

### 验证安装

1. 按 `Win + R`，输入 `cmd` 回车打开命令提示符
2. 输入以下命令验证：

```
node --version
```

   显示 `v18.x.x` 或更高的版本号即表示安装成功

3. 再输入：

```
npm --version
```

   显示 `v9.x.x` 或更高版本号即可

> 💡 如果 `node` 命令提示「不是内部或外部命令」，请重启电脑后再试。

## 快速启动

### 1. 双击启动

桌面双击 **BioAgent.vbs**，会自动完成：

1. 检查 Node.js 是否已安装
2. 安装项目依赖（首次运行自动 `npm install`，耗时约 1-2 分钟）
3. 启动后端服务器（端口 3000）
4. 自动打开浏览器进入监控页面

### 2. 登录页面

首次打开会看到登录弹窗：

```

│             └──────────────────────┘     │
│  端口  ┌────────┐  SSH 用户 ┌────────┐  │

│  💡 首次使用请填写集群信息并「测试连接」  │
└──────────────────────────────────────────┘
```

**使用步骤**：

1. 输入您的**用户名**（平台用户名，用于识别身份）
2. 填写 **Slurm 集群连接信息**：
   - 集群地址（主机名或 IP）
   - 端口（默认 22）
   - SSH 用户名
   - SSH 密码
3. 点击 **「测试连接」** 验证 SSH 连接是否正常
4. 确认连接成功后，点击 **「进入监控」**

登录成功后，用户名会保存在浏览器本地存储中，下次打开自动跳过此页面。

### 3. 修改配置

如需重新配置集群连接，点击顶部导航栏的 ⚙️ 按钮或连接状态区域，即可重新打开登录界面修改。

## 监控面板功能

### 顶部导航栏

```
[← 回到上一级]  🖥 环境生物电化学中心算力集群  [🟢 在线] [⚙] [🔄 刷新] [🧑 用户名 ▾]
```

| 元素 | 说明 |
|---|---|
| **回到上一级** | 跳回 BioAgent 主页面（index.html） |
| **品牌标题** | 显示集群名称 |
| **连接状态** | 🟢 在线 / 🔴 离线 / 🟡 加载中，点击可修改配置 |
| **⚙ 设置** | 打开登录层修改集群连接 |
| **🔄 刷新** | 立即拉取最新集群数据 |
| **用户头像 ▾** | 点击展开用户下拉菜单 |

### 用户下拉菜单

| 项目 | 说明 | 数据来源 |
|---|---|---|
| **运行程序** | 当前运行数 / 总数（含排队） | `squeue` |
| **可用 CPU** | 账户配额可用 CPU 核数 | `sacctmgr assoc` (GrpCPUs) |
| **可用内存** | 账户配额可用内存 | `sacctmgr assoc` (GrpMem) |
| **退出登录** | 清除登录态，回到登录页面 | — |

### 摘要卡片

四张摘要卡片展示集群整体概况：

- **CPU 总量** — 集群总核数、已分配/空闲核数、使用率进度条
- **内存总量** — 集群总内存、已分配/空闲、使用率进度条
- **节点分布** — 总节点数 + 各状态（空闲 / 混合 / 离线）数量
- **我的任务** — 当前用户运行中 + 排队中的任务数

### 存储使用

展示集群各挂载点的磁盘使用情况：

- 挂载点名称、文件系统类型
- 总容量、已用、可用
- 使用率进度条（颜色：🟢 < 60% / 🟡 60-80% / 🔴 > 80%）
- **当前节点内存** — SSH 节点 `free -h` 实时输出
- **用户家目录** — SSH 节点 `du -sh ~` 统计

### 分区

所有 Slurm 分区列表：

- 分区名称
- 已分配节点 / 总节点
- 空闲节点数量
- 使用率进度条

### 节点

每个计算节点的详细信息：

- 节点名称、状态标签（空闲 / 混合 / 满载 / 离线）
- GPU 信息（如分配）
- CPU 分配数、负载
- 内存分配、可用内存
- CPU / 内存使用率进度条
- 归属分区列表

### 我的任务

当前用户在集群中所有任务的表格：

| 列 | 说明 |
|---|---|
| JOBID | 任务编号 |
| 分区 | 所在分区 |
| 名称 | 任务名称 |
| 状态 | 🟢 运行中 / 🟡 排队中 / 🔵 完成中 / 🔴 失败 |
| 运行时间 | 已运行时长 |
| 节点 | 分配节点数 |

## 安全说明

### 密码加密

SSH 密码使用 **AES-256-CBC** 加密后写入配置文件：

| 要素 | 说明 |
|---|---|
| 加密算法 | AES-256-CBC |
| 密钥来源 | 本机 MAC 地址 + 固定 salt → SHA-256 |
| 每次加密 | 随机 16 字节 IV，同一密码每次加密结果不同 |
| 存储文件 | `cluster.config.json`，只存密文，不留明文 |

> **配置文件中永远不会出现密码明文**，即使文件被意外查看也无安全风险。
> 如果配置文件被复制到另一台电脑，因 MAC 地址不同无法解密，需重新输入密码。

### 旧配置迁移

如果 `cluster.config.json` 中仍存在明文的 `password` 字段，服务器会自动加密并覆写文件。

## 故障排除

### 后端未启动

```
❌ 后端服务未运行
```

1. 双击 `BioAgent.vbs` 启动后端
2. 等待命令行窗口显示 `BioAgent Slurm Bridge v2` 字样
3. 刷新页面

### 连接失败

```
❌ SSH 连接失败
```

- 检查集群 IP / 端口是否正确
- 确认 SSH 用户名和密码无误
- 检查集群是否开放 SSH 端口（默认 22）
- 确认网络连通性

### 页面显示异常

- 点击 **「刷新」** 按钮强制刷新数据
- 关闭浏览器重新打开
- 检查后端命令行窗口是否有报错
